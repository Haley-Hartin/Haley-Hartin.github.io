About this app

  • This calculator will take input by clicking on the buttons, or only keyboard input. The keys "a", "w", "s" and "z" will navigate the claculator. The directions are as follows:
    • up: "w" key
    • down: "z" key
    • right: "s" key
    • left: "a" key
    • select: enter/return key
  • To test the UI, I did a test on a 5-key UI and 3-key UI. The 5-key UI is described above. The 3-Key UI uses only the "s" and "a" keys to navigate left/right and the enter/return key to select. The input equations for the experiments are as follows:
    • 5*4+3
    • 10/5-2
    • 7+68-12/2
    • 9/3+4+15-1
    • 7+14/3+2
  • After 4 trials of each UI, here were the test results:
    • 5 button UI- avg time: 2 minutes 44 seconds, errors made: 6
    • 3 button UI- avg time: 2 minutes 16 seconds, errors made: 1
  • Conclusion: I was surprised that the 3 button UI had a quicker completion time on average. In general, a 5 button UI should have a quicker completion time. I think the learning curve for the 5 button UI may have contibuted to its slow completion time. Until you get the hang of it, it takes a while to remember which key is is what. On the other hand, with the 3 button UI, you only have to remember two keys to navigate, so you can react quicker. Over time, I believe the 5 button UI would result in a quicker completion time, but for this limited trial experiment, that was not the case.
